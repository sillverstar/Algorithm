## 1. 탐색(Search)
: 많은 양의 데이터 중에서 원하는 데이터를 찾는 과정

## 2. 자료구조(Data Structure)
: 데이터를 표현하고 관리하고 처리하고 위한 구조
 ex) 스택, 큐

- 삽입(Push): 데이터를 삽입
- 삭제(Pop): 데이터를 삭제

### <스택(Stack)>
    - 선입후출(First In Last Out)
    - 후입선출(Last In First Out; LIFO)

### <큐(Queue)>
    - 선입선출(First In First Out; FIFO)


## 3. 재귀함수(Recursive Function)
: 자기 자신을 다시 호출하는 함수
- 프랙털(Fractal) 구조와 흡사 ex) 시에르핀스티의 삼각형(Sierpinski Triangle)

- 점화식: 특정한 함수를 자신보다 더 작은 변수에 대한 함수와의 관계로 표현한 것


## 4. DFS(Depth-First Search)
: 깊이 우선 탐색이라고도 부르며, 그래프에서 깊은 부분을 우선적으로 탐색하는 알고리즘
-> O(N)

### <DFS의 동작 과정>
    1. 탐색 시작 노드를 스택에 삽입하고 방문 처리를 한다.
    2. 스택의 최상단 노드에 방문하지 않은 인접 노드가 있으면, 그 인접 노드를 스택에 넣고 방문 처리를 한다. 방문하지 않은 인접 노드가 없으면 스택에서 최상단 노드를 꺼낸다.
    3. 2번 과정을 더 이상 수행할 수 없을 때까지 반복한다.

    +) 방문 처리: 스택에 삽입되어 처리된 노드가 다시 삽입되지 않게 처리하는 것

### <그래프(Graph)>
    - 구성: 노드(Node) + 간선(Edge)
    - 두 노드가 간선으로 연결 == 두 노드는 인접하다(Adjacent)
    - 프로그래밍에서의 표현
        1) 인접 행렬(Adjacency Matrix): 2차원 배열로 그래프의 연결관계를 표현하는 방식
        2) 인접 리스트(Adfacency List): 리스트로 그래프의 연결 관계를 표현하는 방식


## 5. BFS(Breadth First Search)
: 너비 우선 탐색이라고도 하며, 가까운 노드부터 탐색하는 알고리즘
-> O(N)

### <BFS의 동작 과정>
    1. 탐색 시작 노드를 큐에 삽입하고 방문 처리를 한다.
    2. 큐에서 노드를 꺼내고, 해당 노드의 인접 노드 중에서 방문하지 않은 노드를 모두 큐에 삽입하고 방문 처리를 한다.
    3. 2번의 과정을 더 이상 수행할 수 없을 때까지 반복한다.
